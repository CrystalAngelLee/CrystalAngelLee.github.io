rollup概述

官网

rollup 是一款 ESModule 的打包器，它也可以将项目中散落的细小模块打包为整块代码，从而使划分的模块更好的运行在浏览器环境orNodejs环境中。

rollup 相比于 webpack 更加小巧，它不支持类似 HMR 这种高级特性。

特点​

  • 自动开启 tree-shaking
  • rollup 默认只能处理ES模块
  • 输出结果更加扁平
  • 打包结果依然完全可阅读
  • ！加载费ESM的第三方模块比较复杂
  • ！模块最终都被打爆到一个函数中，无法实现HMR
  • ！浏览器环境中，代码拆分功能依赖AMD库

快速开始

安装依赖 yarn add rollup --dev

【不常用】使用命令行进行打包指南

配置文件​

  • 在项目根目录下添加文件 rollup.config.js
    • 如果你想要使用 require 和 module.exports 将配置文件写成一个 CommonJS 模块，则应该将文件扩展名更改为 .cjs，这会阻止 Rollup 尝试转译文件
    • 如果你使用的是 Node 13+，则将文件扩展名更改为 .mjs 也可以阻止 Rollup 进行编译，并将文件导入为 ES 模块
  • 配置选项完整列表
  • 使用配置文件在命令行加上--config 或者 -c 的选项

使用插件​

rollup 仅是一款 ESModule 的打包器，如果我们有更高级的需求(比如加载其他类型资源模块、导入CommonJS模块、编译ECMAScript新特性...)，我们就需要使用到插件来扩展 rollup 的功能。

插件是 rollup 唯一扩展途径。

rollup 的插件库：https://github.com/rollup/awesome

// demo 演示 rollup-plugin-json使用
/**
 * 1. 安装依赖 yarn add rollup-plugin-json --dev
 * 2. 添加相关配置到配置文件
 */

// rollup.config.js
import json from 'rollup-plugin-json'

export default {
  input: 'src/index.js',
  output: {
    file: 'dist.js',
    format: 'iife',
  },
  plugins: [json()],
}

// src/index.js  useage
import { name, version } from '../package.json'

console.log(name, version)

加载NPM模块​

rollup 默认只能按照文件路径的方式加载本地文件模块，对于 node_modules 中的第三方模块并不能够通过模块名称导入对应的模块，为了可以提供这个功能，提供了一个rollup-plugin-node-resolve 的插件

  1. 安装插件依赖
  2. 添加插件配置到配置文件

加载 CommonJS 模块

rollup 默认只处理 ESM 的打包，如果在代码中导入CommonJS模块，默认是不被支持的。

  1. 安装插件依赖: yarn add rollup-plugin-commonjs --dev
  2. 添加插件配置到配置文件
